## 二、数据任务

共三个任务，按优先级排序。

---

### 任务一：品类提及率标注（P0）

**目标**：统计 AI 在症状场景（Q3/Q4）中推荐了我方产品所属品类的频率，产出每个产品的"品类提及率"。

**数据来源**：现有 `raw_data.csv`

**操作步骤**：

1. 从 `raw_data.csv` 中筛选 `问题层级` 为 `Q3` 或 `Q4` 的所有行
2. 对每一行，根据该行的 `产品` 字段，在 `应答` 列中检索对应产品的品类关键词（见 1.2 表）
3. 标注结果：命中任一品类关键词 → `品类提及 = 1`，否则 `= 0`
4. 按以下维度汇总统计：`产品 × 模型 × 联网/不联网`

**输出文件**：`category_mention_report.csv`

| 字段名 | 说明 |
|--------|------|
| 产品 | 产品名称 |
| 模型 | 模型名称（DeepSeek / 豆包 / 混元 / 通义千问） |
| 联网 | 是 / 否 |
| Q3Q4总回答数 | 该维度下Q3+Q4的总行数 |
| 品类提及次数 | 其中品类关键词命中的行数 |
| 品类提及率 | 品类提及次数 / Q3Q4总回答数，保留两位小数 |

---

### 任务二：品牌名/通用名分拆统计 + 推荐原因摘录（P0）

**目标**：基于现有数据，将 Q3/Q4/Q5 各层级的应答中，我方品牌名被提及/推荐的情况与通用名被提及/推荐的情况分开统计，并对有推荐的应答摘录推荐原因，供后续分析使用。

**背景**：Q3/Q4 考查症状场景下的无提示提及，Q5 考查品类推荐场景下的无提示提及，三个层级合并来看品牌名与通用名各自的表现。不需要重新采集或修改问题。

**数据来源**：现有 `raw_data.csv`

---

**步骤一：品牌名 / 通用名分别标注**

对 `raw_data.csv` 中 `问题层级` 为 Q3、Q4、Q5 的每一行，在 `应答` 列中分别检索：

- **品牌名命中**：应答中出现 B0 表对应产品的任一品牌关键词 → `品牌名提及 = 1`
- **通用名命中**：应答中出现 B2 表对应产品的通用名 → `通用名提及 = 1`
- 两者互不排斥，同一条应答可以同时命中

**步骤二：推荐标注**

对品牌名提及 = 1 或通用名提及 = 1 的每一行，进一步判断该提及是否属于推荐性提及：

- **推荐 = 1**：AI 将该词作为建议选项、明确推荐或列入推荐清单
- **推荐 = 0**：仅作为举例、对比、警示或否定性提及

此步骤建议使用 LLM 辅助判断，判断逻辑：*"在这条应答中，AI 是否推荐了 [品牌名/通用名]？"*，输出 是/否。

**步骤三：推荐原因摘录**

对推荐 = 1 的每一行，使用 LLM 从应答中摘录 AI 给出的推荐理由，要求：
- 原文摘录或高度贴近原文的改写，不做主观总结
- 长度控制在 50 字以内
- 如应答中未给出理由，填"未说明"

**步骤四：汇总统计**

按 `产品 × 问题层级（Q3/Q4 合并 / Q5）× 模型 × 联网` 维度，分别统计品牌名和通用名的提及率与推荐率。

---

**输出文件一**：`brand_generic_detail.csv`（逐条明细）

| 字段名 | 说明 |
|--------|------|
| 问题ID | 同 raw_data |
| 产品 | 产品名称 |
| 问题层级 | Q3 / Q4 / Q5 |
| 模型 | 模型名称 |
| 联网 | 是 / 否 |
| 品牌名提及 | 0 / 1 |
| 品牌名推荐 | 0 / 1（仅品牌名提及=1时有值） |
| 品牌名推荐原因 | 摘录文本（仅品牌名推荐=1时有值） |
| 通用名提及 | 0 / 1 |
| 通用名推荐 | 0 / 1（仅通用名提及=1时有值） |
| 通用名推荐原因 | 摘录文本（仅通用名推荐=1时有值） |

**输出文件二**：`brand_generic_summary.csv`（汇总统计）

| 字段名 | 说明 |
|--------|------|
| 产品 | 产品名称 |
| 问题层级分组 | Q3/Q4 / Q5 |
| 模型 | 模型名称 |
| 联网 | 是 / 否 |
| 总回答数 | 该维度下的总行数 |
| 品牌名提及率 | 品牌名提及次数 / 总回答数 |
| 品牌名推荐率 | 品牌名推荐次数 / 总回答数 |
| 通用名提及率 | 通用名提及次数 / 总回答数 |
| 通用名推荐率 | 通用名推荐次数 / 总回答数 |

### 任务三：竞品类型分类标注（P1）

**目标**：对现有竞品数据中出现的所有竞品，标注其类型，以便分析中区分"通用名竞争"和"品牌竞争"两个维度。

**数据来源**：现有 `rec_competitor_ranking.csv`

**竞品类型定义（两类）**：

| 类型 | 判断标准 | 举例 |
|------|----------|------|
| 品牌竞品 | 有明确品牌归属的其他厂家产品 | 白加黑、迪巧、双环醇（百赛诺）、美林、泰诺林 |
| 通用名竞品 | 无明确品牌归属的通用名药物，或虽有品牌但以通用名/成分名出现；也包括与我方不同品类但被 AI 同框推荐的药物 | 布洛芬、氯雷他定、奥美拉唑、奥司他韦、他克莫司软膏 |

**操作步骤（AI 先跑样本 → 人工核验 → 确认后批量）**：

由于竞品数量较多且无法穷举，采用 AI 辅助标注，流程如下：

**第一步：AI 标注样本**

从 `rec_competitor_ranking.csv` 中提取所有唯一竞品名，随机抽取 **20 条**，用以下提示词让 AI 逐一标注：

```
任务：将以下药品列表中每个竞品分类为"品牌竞品"或"通用名竞品"。

定义：
- 品牌竞品：有明确品牌归属的其他厂家产品
- 通用名竞品：通用名药物（无明确品牌），或虽有品牌但以通用名/成分名出现的，
  或与目标产品不同品类但被 AI 同框推荐的

已知示例（供参考校准）：
品牌竞品：白加黑、迪巧、百赛诺（双环醇品牌名）、美林、泰诺林
通用名竞品：布洛芬、氯雷他定、奥美拉唑、奥司他韦、他克莫司软膏、氨溴索

请对以下竞品列表逐一标注：
[此处粘贴抽取的 20 条竞品]
```

**第二步：人工核验**

将 AI 标注的 20 条结果发给产品侧/分析侧人工核验：
- 核验无误（≥ 90% 正确）→ 进入第三步
- 核验发现偏差 → 将错误案例补充到提示词示例中，返回第一步重新标注

**第三步：AI 批量标注全量**

核验通过后，用相同提示词对全部竞品批量标注，完成后填入 `rec_competitor_ranking_typed.csv`。

**输出文件**：在原 `rec_competitor_ranking.csv` 基础上增加 `竞品类型` 列并另存为 `rec_competitor_ranking_typed.csv`
