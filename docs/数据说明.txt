一、数据生成流程
================================================================================

脚本执行顺序：
01_parse_questions.py          → 解析xlsx生成基础问题JSON
02_expand_questions.py         → 调用OpenAI生成问题变体
03_query_models.py             → 批量调用国内大模型API回收应答
04_analyze_results.py          → 离线统计分析 + 相似度计算
05_extract_recommendations.py  → LLM提取推荐信息 + 分类统计 + 统一提及率报表
06_build_knowledge_base.py     → 产品资料提取为结构化知识库
07_verify_accuracy.py          → 知识库校对（keyword/llm模式）
08_generate_report.py          → 调用GPT生成完整分析报告（手动触发）

关键配置文件：
config/brands.yaml             → 品牌词典 + V6分析框架关键词配置
  - brand_999                  → 999产品及别名
  - competitors                → 竞品词典
  - product_name_map           → 产品短名→标准名映射（如 小感→小儿感冒颗粒）
  - brand_keywords (B0)        → 999品牌专属关键词（含999/三九等品牌标识）
  - category_keywords (B1)     → 品类关键词（感冒药/止咳药等）
  - generic_names (B2)         → 通用名关键词（行业通用药品名，非999专属）
  - known_brand_competitors    → 已知品牌竞品列表
  - component_keywords         → 成分/品类级关键词（用于过滤非具体产品）

修改brands.yaml后重跑04+05即可更新所有分类结果，无需重跑LLM。

二、04脚本生成的报表（results/analysis/）
================================================================================

04脚本为纯计算，不调用LLM。基于parser.py关键词匹配和统计聚合。

--------------------------------------------------------------------------------
2.1 raw_data.csv — 原始数据总表
--------------------------------------------------------------------------------
作用：所有问答的完整原始数据，方便回查和排错。

字段：
- 问题ID          问题唯一标识，如 ganmaolin_q1_overall
- 产品            产品短名，如 感冒灵、小感、强枇
- 产品代码        产品英文代码，如 ganmaolin
- 问题层级        q1_overall / q2_detail / q3_scenario1 / q4_scenario2 / q5_top3 / negative_safety
- 问题类别        accuracy / mention_recommend / negative_safety
- 是否变体        True/False
- 变体来源        原始问题ID（变体才有值）
- 变体类型        colloquial / no_brand / demographic / alt_angle / negative_safety
- 问题            完整问题文本
- 模型            deepseek / qwen / hunyuan / doubao
- 模型名称        DeepSeek / 通义千问 等
- 联网            是/否
- 轮次            1-5
- 应答            完整应答文本
- 应答字数        应答文本长度
- 响应耗时ms      API响应时间
- 引用源URL       联网模式引用的URL（分号分隔）
- 引用源标题      联网模式引用的标题（分号分隔）
- 引用源数量      引用源条数
- 时间戳          API调用时间

--------------------------------------------------------------------------------
2.2 stability_report.csv — 回答稳定性分析
--------------------------------------------------------------------------------
作用：衡量同一问题多轮回答的品牌提及一致性。由05脚本覆盖更新。

字段：
- 问题ID、产品、模型、联网、轮次数
- 推荐产品Jaccard均值   多轮推荐产品集合的平均Jaccard相似度（0-1）
- 推荐排序完全一致      多轮的推荐排序是否完全相同
- 排序变体数            不同排序的种类数
- 999提及稳定性         稳定提及 / 稳定未提及 / 不稳定(N/5轮提及)

--------------------------------------------------------------------------------
2.3 search_diff_report.csv — 联网vs不联网差异分析
--------------------------------------------------------------------------------
作用：对比同一问题联网和不联网下的推荐差异。由05脚本覆盖更新。

字段：
- 问题ID、产品、模型
- 联网999提及率 / 不联网999提及率 / 提及率差异
- 仅联网推荐的产品 / 仅不联网推荐的产品

--------------------------------------------------------------------------------
2.4 source_report.csv — 信息源域名汇总
--------------------------------------------------------------------------------
作用：联网模式下模型引用了哪些网站，按域名统计频次。

字段：域名、引用次数

--------------------------------------------------------------------------------
2.5 source_detail_report.csv — 信息源明细
--------------------------------------------------------------------------------
作用：联网模式下每条引用的详细信息。

字段：问题ID、产品、模型、域名、标题、URL

--------------------------------------------------------------------------------
2.6 competitor_report.csv — 竞品图谱
--------------------------------------------------------------------------------
作用：各产品场景下被AI提及最多的竞品。由05脚本覆盖更新。

字段：产品、竞品、出现次数、出现率

--------------------------------------------------------------------------------
2.7 variant_sensitivity_report.csv — 问法敏感度分析
--------------------------------------------------------------------------------
作用：对比原始问题和变体问题的999提及率差异。

字段：基础问题ID、产品、模型、联网、原始999提及率、变体999提及率、差异

--------------------------------------------------------------------------------
2.8 accuracy_summary.csv — 准确率汇总（供人工评分）
--------------------------------------------------------------------------------
作用：Q1-Q2应答的人工评分模板。

字段：问题ID、产品、模型、联网、轮次、问题、应答、人工评分（空）、备注（空）

--------------------------------------------------------------------------------
2.9 cross_model_consistency.csv — 跨模型一致性分析
--------------------------------------------------------------------------------
作用：同一问题在不同模型间的品牌提及一致性。

字段：问题ID、产品、联网、模型数、品牌并集数、品牌交集数、品牌一致度、
      各模型一致提及、{model}_999提及率（动态列）

--------------------------------------------------------------------------------
2.10 optimization_report.csv — 优化建议
--------------------------------------------------------------------------------
作用：自动诊断各产品×模型的薄弱环节。由05脚本覆盖更新。

字段：产品、模型、优先级（高/中/低）、问题数、发现、主要竞品、建议方向

--------------------------------------------------------------------------------
2.11 unknown_brands.csv — 未收录品牌候选
--------------------------------------------------------------------------------
作用：词典中未收录但反复出现的药品名，供审核补充到brands.yaml。

字段：候选品牌名、出现次数（≥2次才输出）

--------------------------------------------------------------------------------
2.12 answer_similarity.csv — 多轮回答相似度
--------------------------------------------------------------------------------
作用：TF-IDF衡量同一问题多轮回答的文本相似度，供LLM校对时决定发送策略。

字段：问题ID、产品、模型、联网、轮次数、平均字数、字数标准差、
      TF-IDF相似度、LLM校对建议策略、问题

三、05脚本生成的报表（results/analysis/）
================================================================================

05脚本先用LLM（DeepSeek）提取推荐信息（支持断点，已跑完自动跳过），
然后基于提取结果做纯计算的统计分析和分类。
分类逻辑全部依赖brands.yaml配置，修改yaml重跑即可。

--------------------------------------------------------------------------------
3.1 recommendation_detail.csv — 推荐提取明细（LLM生成）
--------------------------------------------------------------------------------
作用：LLM从每条Q3-Q5应答中提取的结构化推荐信息。所有后续报表的数据源。
生成：LLM提取（DeepSeek V3.2），支持断点续跑

字段：
- 问题ID          问题唯一标识
- 产品            场景所属的产品短名
- 模型            原始回答的模型
- 联网            是/否
- 轮次            1-5
- 推荐排名        该产品在推荐中的排名（1为最强）
- 推荐产品        被推荐的产品名称（保持原文写法）
- 推荐强度        strong(强推荐) / moderate(可选) / mention(仅提及) / caution(警告)
- 推荐理由        从原文提取的推荐理由（30字以内）
- 品类            产品品类（LLM判断）

--------------------------------------------------------------------------------
3.2 brand_generic_detail.csv — 推荐产品分类明细
--------------------------------------------------------------------------------
作用：对recommendation_detail中每条推荐产品标注名称类型，覆盖全部Q3-Q5数据。
生成：纯关键词匹配（基于brands.yaml配置）

名称类型分类逻辑（优先级从高到低）：
1. 999品牌     — 推荐产品命中 brand_keywords (B0)，含999/三九等品牌标识
2. 竞品品牌    — 推荐产品命中 known_brand_competitors
3. 成分品类级  — 推荐产品命中 component_keywords，不是具体产品
4. 通用名      — 以上都未命中，为行业通用药品名（可能是任何厂家的产品）

字段：
- 问题ID、产品、问题层级（Q3/Q4/Q5）、模型、联网、轮次
- 推荐排名        原始推荐排名
- 推荐产品        产品名称原文
- 名称类型        999品牌 / 竞品品牌 / 通用名 / 成分品类级
- 推荐强度        strong / moderate / mention / caution
- 是否推荐        1(strong/moderate) / 0(mention/caution)
- 推荐原因        推荐理由（仅是否推荐=1时有值）

--------------------------------------------------------------------------------
3.3 mention_report.csv — 统一提及率报表
--------------------------------------------------------------------------------
作用：合并品类提及、999品牌、通用名、竞品品牌的提及数据，一张表看全貌。
生成：品类提及从原始应答文本匹配，其余从recommendation_detail分类结果统计

维度：产品 × 问题层级(Q3/Q4 或 Q5) × 模型 × 联网

字段：
- 产品、问题层级、模型、联网、总回答数
- 品类提及率           Q3/Q4的原始应答中品类关键词(B1)命中率，Q5留空
- 999品牌提及次数/率   推荐产品命中B0品牌关键词的次数和比率
- 999品牌推荐率        其中strong/moderate的比率
- 通用名提及次数/率    通用名类推荐产品的次数和比率
- 通用名推荐率         其中strong/moderate的比率
- 竞品品牌提及次数/率  竞品品牌类推荐产品的次数和比率
- 竞品品牌推荐率       其中strong/moderate的比率
- 成分品类级提及次数   成分/品类级推荐产品的次数

注：提及次数可能>总回答数，因为一条应答中LLM可能提取出多个推荐产品。

--------------------------------------------------------------------------------
3.4 brand_generic_summary.csv — 推荐产品分类汇总
--------------------------------------------------------------------------------
作用：与mention_report.csv内容相同，保留供交叉验证。

--------------------------------------------------------------------------------
3.5 recommendation_extracted_summary.csv — 推荐提取汇总
--------------------------------------------------------------------------------
作用：按场景产品×模型×被推荐产品汇总各推荐强度的出现次数。

字段：场景产品、模型、被推荐产品、强推荐次数、可选次数、仅提及次数、
      警告次数、总出现次数

--------------------------------------------------------------------------------
3.6 rec_999_overview.csv — 999推荐总览
--------------------------------------------------------------------------------
作用：每个产品×模型×联网下，999的推荐情况总览。

字段：产品、模型、联网、应答总数、999强推荐次数/率、999被提及次数/率、
      999平均排名、999最佳排名

--------------------------------------------------------------------------------
3.7 rec_competitor_ranking.csv — 竞品推荐排行
--------------------------------------------------------------------------------
作用：每个场景×模型下被推荐最多的Top15竞品。

字段：场景产品、模型、排名、竞品、强推荐次数、可选次数、总出现次数

--------------------------------------------------------------------------------
3.8 rec_competitor_ranking_typed.csv — 竞品推荐排行（含类型标注）
--------------------------------------------------------------------------------
作用：在rec_competitor_ranking基础上新增竞品类型列。
生成：基于brands.yaml的known_brand_competitors和component_keywords分类

竞品类型分类逻辑：
1. 品牌竞品     — 命中 known_brand_competitors
2. 成分/品类级  — 命中 component_keywords
3. 通用名竞品   — 以上都未命中（默认）

字段：同rec_competitor_ranking + 竞品类型（品牌竞品/通用名竞品/成分/品类级）

--------------------------------------------------------------------------------
3.9 rec_reasons.csv — 推荐理由分析
--------------------------------------------------------------------------------
作用：999产品和竞品各自被推荐的高频理由。

字段：类型（999产品/竞品）、推荐理由、出现次数（各取Top30）

--------------------------------------------------------------------------------
3.10 rec_search_impact.csv — 联网影响分析
--------------------------------------------------------------------------------
作用：对比联网和不联网下999产品的推荐变化。

字段：产品、模型、不联网_强推荐率、联网_强推荐率、强推荐率变化、
      不联网_提及率、联网_提及率、提及率变化、不联网_平均排名、联网_平均排名

--------------------------------------------------------------------------------
3.11 dashboard.csv — 总览Dashboard（05脚本覆盖更新）
--------------------------------------------------------------------------------
作用：全局核心指标一张表，基于Q3-Q5 LLM提取数据。

字段：产品、模型、Q3-Q5应答总数、
      999强推荐率(整体/不联网/联网)、999提及率(整体/不联网/联网)、
      999平均排名、999最佳排名、Top3竞品

四、07脚本生成的报表（results/analysis/）
================================================================================

--------------------------------------------------------------------------------
4.1 accuracy_keyword_verify.csv — 关键词校对
--------------------------------------------------------------------------------
作用：用知识库关键词检查AI回答的信息覆盖情况。免费离线。
生成：07脚本 --mode keyword

字段：问题ID、产品、模型、联网、轮次、关键知识点总数、覆盖数、覆盖率、
      发现错误（是/否）、错误内容

--------------------------------------------------------------------------------
4.2 accuracy_llm_verify.csv — LLM校对明细
--------------------------------------------------------------------------------
作用：DeepSeek对每个知识点的逐条校对结果，综合多轮回答判断。
生成：07脚本 --mode llm

字段：question_id、product、model、search_enabled、round_count、
      fact_index、fact、importance、category、
      verdict（correct/wrong/missing/inconsistent/not_applicable）、
      correct_rounds、wrong_rounds、detail、answer_similarity、send_strategy

--------------------------------------------------------------------------------
4.3 accuracy_llm_summary.csv — LLM校对汇总
--------------------------------------------------------------------------------
作用：按产品×模型汇总LLM校对结果。
生成：07脚本 --mode llm

字段：产品、模型、校对知识点数、正确、错误、遗漏、不稳定、
      准确率、完整率、稳定率

六、建议分析顺序
================================================================================

1. 全局概览     dashboard.csv → 一张表定位全局强弱
2. 提及分析     mention_report.csv → 品类/品牌/通用名/竞品提及全貌
3. 推荐详情     rec_999_overview → rec_competitor_ranking_typed → rec_reasons
4. 分类下钻     brand_generic_detail.csv → 逐条查看推荐产品分类
5. 联网影响     rec_search_impact → search_diff_report → source_detail_report
6. 稳定性       stability_report → answer_similarity
7. 问法敏感度   variant_sensitivity_report
8. 跨模型对比   cross_model_consistency
9. 准确率校对   accuracy_keyword_verify → accuracy_llm_summary
10. 优化建议    optimization_report
11. 完整报告    运行08脚本生成AI分析报告

================================================================================

七、关键词分类校准流程
================================================================================

如发现分类不准确：

1. 编辑 config/brands.yaml 中对应配置节
   - 999品牌词不对  → 改 brand_keywords
   - 品类词不对     → 改 category_keywords
   - 通用名不对     → 改 generic_names
   - 竞品分类不对   → 改 known_brand_competitors
   - 成分词误判     → 改 component_keywords
   - 产品名映射缺失 → 改 product_name_map

2. 重跑（不会触发LLM）
   python scripts/04_analyze_results.py
   python scripts/05_extract_recommendations.py

3. 检查结果，重复以上步骤直到满意
