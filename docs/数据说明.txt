一、数据生成流程
================================================================================

脚本执行顺序：
01_parse_questions.py     → 解析xlsx生成基础问题JSON
02_expand_questions.py    → 调用OpenAI生成问题变体
03_query_models.py        → 批量调用国内大模型API回收应答
04_analyze_results.py     → 离线统计分析 + 相似度计算
05_generate_report.py     → 调用GPT-5.4生成完整分析报告（手动触发）
06_build_knowledge_base.py → 产品资料提取为结构化知识库
07_verify_accuracy.py     → 知识库校对（keyword/llm模式）
08_extract_recommendations.py → LLM提取推荐信息 + 统计分析 + 更新Dashboard

--------------------------------------------------------------------------------
2.1 raw_data.csv — 原始数据总表
--------------------------------------------------------------------------------
作用：所有问答的完整原始数据，方便回查和排错。
生成：04脚本

字段：
- 问题ID          问题唯一标识，如 ganmaolin_q1_overall
- 产品            产品名称，如 感冒灵
- 产品代码        产品英文代码，如 ganmaolin
- 问题层级        q1_overall / q2_detail / q3_scenario1 / q4_scenario2 / q5_top3 / negative_safety
- 问题类别        accuracy（准确率类）/ mention_recommend（提及推荐类）/ negative_safety（负面安全类）
- 是否变体        True/False，是否为问题变体
- 变体来源        原始问题ID（变体才有值）
- 变体类型        colloquial(口语化) / no_brand(去品牌) / demographic(人群细化) / alt_angle(换角度) / negative_safety
- 问题            完整问题文本
- 模型            模型标识，如 deepseek / qwen / hunyuan / doubao
- 模型名称        模型中文名，如 DeepSeek / 通义千问
- 联网            是/否
- 轮次            1-5
- 应答            完整应答文本
- 应答字数        应答文本长度
- 响应耗时ms      API响应时间（毫秒）
- 引用源URL       联网模式下引用的网页URL（分号分隔）
- 引用源标题      联网模式下引用的网页标题（分号分隔）
- 引用源数量      引用源条数
- 时间戳          API调用时间

2.12 optimization_report.csv — 优化建议
--------------------------------------------------------------------------------
作用：自动诊断每个产品在每个模型上的薄弱环节，输出优先级和建议方向。
生成：04脚本

字段：
- 产品            999产品名称
- 模型            模型标识
- 优先级          高/中/低
- 问题数          发现的问题数量
- 发现            具体问题描述（多条用|分隔）
- 主要竞品        该场景下出现最多的竞品
- 建议方向        对应的优化建议（多条用|分隔）

判断逻辑：
- 场景题999提及率<30% → 高优先级
- Top3完全没进 → 高优先级
- 联网后提及率下降>20% → 高优先级
- 多轮回答不稳定>40% → 中优先级
- 进Top3但排名靠后>2.0 → 中优先级

4.1 accuracy_keyword_verify.csv — 关键词校对
--------------------------------------------------------------------------------
作用：用知识库关键词快速检查AI回答对产品信息的覆盖情况。免费离线，秒出结果。
生成：07脚本 --mode keyword

字段：
- 问题ID          问题唯一标识
- 产品            999产品名称
- 模型            模型标识
- 联网            是/否
- 轮次            轮次编号
- 关键知识点总数  critical级别的知识点数量
- 覆盖数          关键词覆盖率>50%的知识点数
- 覆盖率          覆盖数 / 关键知识点总数
- 发现错误        是/否，是否出现已知的错误说法
- 错误内容        具体的错误说法内容

--------------------------------------------------------------------------------
4.2 accuracy_llm_verify.csv — LLM校对明细
--------------------------------------------------------------------------------
作用：GPT-5.4对每个知识点的逐条校对结果。综合多轮回答判断。
生成：07脚本 --mode llm

字段：
- question_id      问题唯一标识
- product          999产品名称
- model            模型标识
- search_enabled   是否联网
- round_count      聚合的轮次数
- fact_index       知识点序号
- fact             知识点内容
- importance       知识点重要程度（critical/important）
- category         知识点类别（efficacy/ingredients/contraindication等）
- verdict          判定结果：
                   - correct: 多数轮次正确提及
                   - wrong: 存在与知识点矛盾的回答
                   - missing: 所有轮次均未提及
                   - inconsistent: 部分轮次正确、部分遗漏
                   - not_applicable: 与问题无关
- correct_rounds   正确提及的轮次数
- wrong_rounds     出现错误的轮次数
- detail           简要说明
- answer_similarity 该问题多轮回答的TF-IDF相似度
- send_strategy    发送策略（取2条代表/全量发送）

--------------------------------------------------------------------------------
4.3 accuracy_llm_summary.csv — LLM校对汇总
--------------------------------------------------------------------------------
作用：按产品×模型汇总LLM校对结果。
生成：07脚本 --mode llm

字段：
- 产品            999产品名称
- 模型            模型标识
- 校对知识点数    参与校对的知识点总数（排除not_applicable）
- 正确            verdict=correct的数量
- 错误            verdict=wrong的数量
- 遗漏            verdict=missing的数量
- 不稳定          verdict=inconsistent的数量
- 准确率          正确 / (正确 + 错误)
- 完整率          正确 / 校对知识点数
- 稳定率          (正确 + 遗漏) / 校对知识点数（排除inconsistent的比例）

六、建议分析顺序
================================================================================

1. 全局概览     dashboard.csv → 一张表定位全局强弱
2. 推荐分析     rec_999_overview → rec_competitor_ranking → rec_search_impact → rec_reasons
3. 提及与认知   mention_report → cross_model_consistency → competitor_report
4. 稳定性       answer_similarity → stability_report
5. 问法敏感度   variant_sensitivity_report
6. 联网深挖     search_diff_report → source_report → source_detail_report
7. 准确率校对   accuracy_summary → accuracy_keyword_verify → accuracy_llm_summary
8. 优化建议     optimization_report
9. 完整报告     运行05脚本生成AI分析报告
