# GEO分析待补充数据说明

> 本文件列出为完整支撑V6分析框架所需补充的数据项，分为两类：
> - **A类**：需要重新跑取或标注的数据（由数据分析侧处理）
> - **B类**：需要各产品负责人提供的基础信息（由产品侧提供）
>
> 数据补充完成后，V6框架中的【待补充】项将更新，产品分组也可能随之调整。

## A类：需重新跑取或标注的数据

### A1. 品类提及数据（病症阶段核心指标）

**用途**：判断AI在症状场景是否推荐我们的药品品类，是第一组分组的核心判断依据。

**当前缺口**：现有数据只记录了品牌/通用名的提及情况，未记录AI推荐了哪些药品品类。

**补充方案（二选一）**：

方案A — 重新标注现有Q3/Q4原始回答
- 从raw_data.csv中筛选Q3/Q4层级的所有回答
- 针对每个产品预先定义"品类关键词"（见B类信息）
- 逐条标注该回答中是否出现品类关键词
- 统计品类关键词出现率 = 品类提及次数 / Q3/Q4总回答数

方案B — 新增品类维度问题
- 设计以"品类"而非"品牌"为中心的Q3/Q4问题（如"感冒了该选哪类药"）
- 重新采集并统计品类推荐分布

**需要先提供**：各产品的品类关键词（见B1）

**预期产出**：每个产品的"品类提及率"数值，区分模型和联网/不联网模式

---

### A2. 品牌名/通用名分拆数据（品类阶段核心指标）

**用途**：区分AI推荐"感冒灵颗粒"（通用名）vs"999感冒灵"（品牌名）的差异，是品类阶段分析的核心数据。

**当前缺口**：现有Q5数据中品牌名与通用名混合，无法分开统计。

**补充方案**：
- 每个产品分两组重新跑Q5问题：
  - **品牌名组**：问题中出现品牌名（如"999感冒灵好不好"、"推荐感冒药，999感冒灵怎么样"）
  - **通用名组**：问题中出现通用名（如"感冒灵颗粒好不好"、"推荐感冒药，感冒灵颗粒怎么样"）
- 分别统计提及率、强推荐率、平均排名

**需要先提供**：各产品的正式通用名（见B2）

**预期产出**：

| 指标 | 品牌名组 | 通用名组 |
|------|--------|--------|
| 提及率 | X% | X% |
| 强推荐率 | X% | X% |
| 平均排名 | X | X |

---

### A3. 负向内容结构化数据（品牌阶段核心指标）

**用途**：量化AI在提及品牌时附带负向内容的频率，是第三组产品分析的关键指标。

**当前缺口**：现有rec_reasons.csv中有负向内容记录，但：
- 未区分发生在哪个问题层级（Q1/Q2 vs Q3/Q4 vs Q5）
- 未统计负向提及率（负向提及次数/总提及次数）
- 未标注负向内容的严重程度

**补充方案**：

步骤1 — 从raw_data.csv中筛选出包含品牌名/通用名提及的所有回答

步骤2 — 对每条提及回答进行负向标注（可用关键词规则或LLM辅助）：
- **是否含负向内容**：有/无
- **负向内容严重程度**：🔴强禁止性（"绝对禁忌"/"不要用"）/ 🟡限制性（"注意"/"慎用"）
- **发生的问题层级**：Q1/Q2 / Q3/Q4 / Q5

步骤3 — 统计：
- 负向提及率 = 含负向内容的提及次数 / 总提及次数（按产品×模型×问题层级）
- 负向内容高频词汇汇总

**预期产出**：

| 产品 | Q1/Q2负向提及率 | Q3/Q4负向提及率 | Q5负向提及率 | 主要负向内容 |
|------|:----------:|:----------:|:--------:|-----------|
| 皮炎平 | X% | X% | X% | "含强效激素…" |
| 强枇 | X% | X% | X% | "含罂粟壳…" |
| … | | | | |

### A4. 竞品类型分类标注（品类阶段辅助指标）

**用途**：区分品类层的竞品是通用名竞品还是品牌竞品，用于更精准的竞争格局分析。

**当前缺口**：rec_competitor_ranking.csv中的竞品未标注类型，无法做分类对比。

**补充方案**：
- 对rec_competitor_ranking.csv中出现的所有竞品，逐一标注：
  - **通用名竞品**：同品类的通用名药物（如氯雷他定、布洛芬）
  - **品牌竞品**：同品类的其他品牌产品（如白加黑、迪巧）
  - **跨品类竞品**：不同品类但在AI中与我方产品同框竞争的产品（如奥司他韦之于抗病毒口服液）
- 重新计算各类竞品的提及率，与我方产品对比

**预期产出**：结构化竞品对比表，区分"通用名 vs 通用名"和"品牌 vs 品牌"两个维度的差距

---

### A5. 知识准确率LLM校对（品牌阶段辅助指标，可选）

**用途**：精确判断AI对品牌知识点描述的准确程度，当前关键词匹配版精度有限。

**当前缺口**：accuracy_llm_summary.csv和accuracy_llm_verify.csv尚未生成，只有关键词版本（accuracy_keyword_verify.csv）。

**补充方案**：
- 运行07脚本的LLM模式（`python 07_verify_accuracy.py --mode llm`）
- 生成accuracy_llm_summary.csv（按产品×模型汇总）和accuracy_llm_verify.csv（逐知识点明细）

**建议优先产品**：皮炎平、强枇（负向内容最严重）；其次感冒灵、小感

**预期产出**：各产品的准确率、遗漏率、错误率、不稳定率
