"先看全局 → 再看细节 → 最后看优化方向"的逻辑：

  第一步：全局概览
                                                                                                                                                                     raw_data.csv → 先扫一眼数据量是否完整，各模型×产品×联网模式的覆盖是否齐全

  dashboard.csv → 一张表看所有核心指标，快速定位哪些产品×模型组合表现好/差

  第二步：推荐分析（最核心）

  rec_999_overview.csv → 999在各场景下被推荐的情况，强推荐率和提及率是多少

  rec_competitor_ranking.csv → 每个场景下谁是999的主要对手，竞品被推荐的频次

  rec_search_impact.csv → 联网后999的推荐是变好还是变差

  rec_reasons.csv → 模型推荐999和竞品的理由分别是什么，能看出品牌认知差异

  第三步：提及与认知

  mention_report.csv → 按问题层级看提及率，区分"模型认不认识你"（q1-q2）和"模型会不会推荐你"（q3-q5）

  cross_model_consistency.csv → 各模型对同一问题的品牌提及是否一致，分歧大的问题值得深挖

  competitor_report.csv → 各品类的竞品全景图

  第四步：稳定性与敏感度

  answer_similarity.csv → 多轮回答相似度，低相似度说明模型输出不稳定

  stability_report.csv → 品牌提及的跨轮一致性

  variant_sensitivity_report.csv → 换一种问法结果变多少，敏感度高=优化切入点

  第五步：联网深挖

  search_diff_report.csv → 联网vs不联网的详细对比

  source_report.csv + source_detail_report.csv → 联网时模型参考了哪些网站，这直接指向后续SEO优化的目标

  第六步：准确率（需要知识库）

  accuracy_summary.csv → 人工审核or喂给LLM校对

  accuracy_keyword_verify.csv → 关键词覆盖情况

  accuracy_llm_verify.csv / accuracy_llm_summary.csv → LLM精准校对结果

  第七步：汇总输出

  optimization_report.csv → 自动诊断的优化建议

  05_generate_report.py → 把以上数据喂给GPT-5.4，生成完整分析报告
